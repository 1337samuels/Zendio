Build a full-stack web app called RemitRadar — "Google Flights for remittances."

It helps diaspora workers find the cheapest multi-hop route to send money between countries. All data is mock/static for now (no real APIs). The app has 4 pages: Route Finder, Rate Tracker, Smart Timing, and Rate Alerts.

## Tech Stack
- Frontend: React with Vite, TypeScript, Tailwind CSS, shadcn/ui, recharts for charts
- Backend: Express.js API serving mock data from JSON files
- Use React Router for page navigation
- Framer Motion for animations

## Design System

Theme: dark fintech aesthetic, premium but approachable.

CSS variables:
--bg-primary: #0F1729 (deep navy)
--bg-card: rgba(255, 255, 255, 0.04) with backdrop-blur-xl
--accent-teal: #00D4AA (primary actions, best route highlights)
--accent-amber: #F5A623 (alerts, warnings)
--accent-red: #FF6B6B (expensive routes, bad rates)
--text-primary: #F0F0F0
--text-muted: #8A9BB5

Typography: Import "DM Sans" (body) and "Bricolage Grotesque" (headings) from Google Fonts. Never use Inter, Roboto, or system fonts.

Cards: rounded-xl with bg-white/5 backdrop-blur-xl border border-white/10. On hover, subtle scale and border-white/20 transition.

- Total cost in destination currency, large bold text (e.g., "£11.20")

C) The cheapest card has:
- Teal border glow (box-shadow: 0 0 20px rgba(0, 212, 170, 0.15))

D) Non-cheapest cards show: "You'd pay £XX.XX more" in muted red below the total cost

## Page 3: Smart Timing (the differentiator)

This page shows that the REAL cost of remittance includes both FX and platform fees, and that the cheapest route changes over time.

Top: corridor selector + explanatory subtitle in muted text: "Raw exchange rates don't tell the full story. Platform fees and spreads change independently. We track the real cost of your optimal route over time."

Main chart — Recharts LineChart with TWO lines:
- Line 1 (gray, dashed, label "FX Rate Only"): raw exchange rate over 90 days (same data as Rate Tracker, but normalize to show as cost percentage, e.g., deviation from best possible rate)
- Line 2 (teal, solid, thick, label "True Cost via Best Route"): total cost percentage including FX + all hop fees. Generate this so it generally tracks Line 1 but diverges at key moments.

Create 3 annotation points on the chart where the lines diverge, rendered as small marker dots with tooltips:
- Day ~30: True Cost dips to 0.8% while FX was average. Tooltip: "Binance P2P spread dropped — crypto route was cheapest this week"
- Day ~60: FX rate was favorable but True Cost didn't improve much. Tooltip: "Wise increased fees — offset the good FX rate"  
- Day ~75: Both lines hit their minimum. Tooltip: "Best day to send — 0.6% total cost"

Insight cards below chart:

Card 1 — "Route shifts this month" with a vertical mini timeline:
- "Feb 1–8: Dollar App route was cheapest (0.9% avg)"

- "Feb 16–today: Dollar App route is cheapest again (0.85%)"
